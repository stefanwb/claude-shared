## 2. Smoke tests against scenarios

- [x] 2.1 No-flag invariant: with a host `~/.npmrc` setting a private `registry=` and `UV_DEFAULT_INDEX` exported, `claude-docker ~/repo` (no flag) ŌåÆ inside the container `/root/.npmrc` lacks the host config, `echo $UV_DEFAULT_INDEX` is empty, and `pnpm config get registry` returns the public npm default
- [x] 2.2 `--registry` npmrc mount: with a host `~/.npmrc` pointing at a private registry, `claude-docker --registry ~/repo` exposes it read-only ŌĆö `pnpm config get registry` returns the private URL and an in-container write to `/root/.npmrc` fails with EROFS
- [x] 2.3 `--registry` uv env forwarding incl. dynamic vars: with `UV_DEFAULT_INDEX`, `UV_INDEX_INTERNAL_USERNAME`, and `UV_INDEX_INTERNAL_PASSWORD` exported on the host, `claude-docker --registry ~/repo` forwards all three; without `--registry`, none are present
- [x] 2.4 `--registry` silent absence: with no host `~/.npmrc`/`~/.netrc`/`uv.toml` and none of the forwarded env vars set, `claude-docker --registry ~/repo` starts cleanly with no error and the package managers use public defaults
- [x] 2.5 Statusline tag includes `registry`: launch `claude-docker --registry --gh ~/repo` and confirm the statusline prefix shows `registry` in the `docker:` tag list (e.g. `docker:gh,registry`)
- [x] 2.6 End-to-end (CodeArtifact, manual): on a host with `aws codeartifact login --tool npm` having written a token into `~/.npmrc`, `claude-docker --registry ~/repo` ŌåÆ `pnpm add <a-package-in-the-feed>` resolves via the private registry
- [x] 2.7 `--registry` pip config: with a host `pip.conf` (platform-aware path) setting a private `index-url`, `claude-docker --registry ~/repo` exposes it read-only at `/root/.config/pip/pip.conf`; verify via `uvx pipenv` (or `uv pip --no-config`-bypassed check) that pip-side resolution targets the private index; confirm `PIP_INDEX_URL`/`PIPENV_PYPI_MIRROR` forward only under the flag
- [x] 2.8 End-to-end (pip via uvx, manual): on a host with `aws codeartifact login --tool pip` having written `index-url` into `pip.conf`, `claude-docker --registry ~/repo` ŌåÆ `uvx pipenv install <a-package-in-the-feed>` resolves via the private index (documents the no-bundled-Python path from D8)
